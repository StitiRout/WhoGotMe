# WhoGotMe
This is a cybersecurity-focused web application designed to detect whether a user’s email address or phone number has been exposed in data breaches. 

The application allows users to input their email or phone number, which is processed temporarily in memory and checked against a simulated breach database. The system then displays relevant information such as breach sources, dates, and a calculated risk level, along with recommended security actions like password changes and enabling two-factor authentication.

A key feature of the project is its strong emphasis on user privacy and security. The system ensures that user inputs are never stored permanently, avoids logging sensitive data, and optionally uses hashing techniques to protect stored breach records. This makes the application both secure and ethically compliant.

The project is built using a modern tech stack including Java (Spring Boot) for backend development, HTML/CSS/JavaScript for the frontend, and MongoDB for database management. Additional security measures such as input validation, rate limiting, and HTTPS communication enhance the system’s robustness.

Overall, this project demonstrates practical knowledge of cybersecurity concepts, secure coding practices, and web application development, while addressing a real-world problem of increasing data breaches and user awareness.
